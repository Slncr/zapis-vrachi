# Ошибка 500 при создании записи (AppointmentCreate)

При нажатии «Подтвердить» бот отправляет POST на `.../hs/bwi/AppointmentCreate` с методом **BookAnAppointmentWithParams**. Ответ 1С: **500** с текстом:

```
Не удалось записать документ: Ошибка при вызове метода контекста (Записать): Не удалось записать: "Заявка"!
```

## Что проверить в 1С

1. **Документ «Заявка»**  
   В конфигурации должен быть документ с именем «Заявка», и у пользователя МИС (логин/пароль из настроек бота) должны быть права на создание и запись этого документа.

2. **Обязательные реквизиты**  
   При записи документа 1С может падать, если не заполнены обязательные реквизиты или не пройдена проверка (например, дата/время приёма, сотрудник, клиника, пациент). Проверьте обработчик метода записи и обработку в модуле документа «Заявка».

3. **Формат параметров запроса**  
   Бот отправляет JSON:
   - `Date` — дата приёма в формате `yyyyMMddT00:00:00` (например, `20260201T00:00:00`)
   - `TimeBegin` — время начала в формате `HH:MM:SS` (например, `11:30:00`)
   - `Birthday` — дата рождения в формате `YYYY-MM-DD`
   - `EmployeeID`, `Clinic` — UUID (строки)
   - `PatientSurname`, `PatientName`, `PatientFatherName`, `Phone` — строки

4. **Логи 1С**  
   В логе 1С (или журнале регистрации) при воспроизведении ошибки должна быть подробная причина (какой реквизит не прошёл проверку, какое условие не выполнилось).

5. **Сравнение с N8N**  
   Если запись из N8N проходила успешно, сравните тело запроса из N8N с тем, что отправляет бот (в логах бота пишется `AppointmentCreate body: Date=..., TimeBegin=..., Clinic=..., EmployeeID=...`).

## Контакты

После проверки выше при необходимости можно скорректировать формат полей в коде бота (см. `app/mis_client.py`, метод `create_appointment`).
