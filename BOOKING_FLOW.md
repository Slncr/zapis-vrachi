# Поток записи после выбора врача

Ниже актуальная схема, как сейчас работает запись пациента в боте (Telegram/MAX), и какие методы МИС участвуют.

## 1) Выбор филиала врача

После регистрации врача (ФИО + телефон) бот сразу открывает список филиалов, где принимает врач.

Источник данных:
- локальная БД `doctors.clinic_uids` (заполняется из расписания МИС)
- отображаемые имена филиалов:
  - `clinics` в БД
  - + `filials_uid.txt` (приоритет для человекочитаемого названия)

## 2) Календарь по выбранному филиалу

После выбора филиала бот показывает календарь месяца, где `*` = у выбранного врача есть расписание в этот день в этом филиале.

Метод МИС:
- `Schedule` (через `get_schedule20(...)`)
- эндпоинт: `.../hs/bwi/Schedule`
- в запросе: `Employee`, `StartDate`, `FinishDate`, `Key`, `User`, `Password`
- дополнительно в коде стоит локальная фильтрация по `СотрудникID`, потому что контур иногда возвращает общий график.

## 3) Дневное расписание (по клику на дату)

При выборе дня бот показывает:
- занятые записи (время + пациент + услуга),
- свободные слоты кнопками.

Методы МИС:
- `Schedule` — интервалы свободного/занятого времени (`СвободноеВремя`, `ЗанятоеВремя`)
- `PatientTickets` — пациент и услуга для занятых слотов:
  - `КлиентНаименование`
  - `СписокРабот[].Наименование`
- запрос `PatientTickets` идет по расширенному окну дат, затем локальная фильтрация по:
  - дню,
  - филиалу,
  - врачу.

## 4) Выбор свободного слота

После нажатия на свободное время бот переходит к выбору услуги врача.

## 5) Выбор услуги врача

Бот показывает услуги (по одной кнопке в строке).

Источник услуг:
1. сначала `doctors.main_services` из БД (кэш)
2. если пусто — on-demand из `PatientTickets` (парсится `СписокРабот`)
3. найденные услуги кэшируются в БД (`set_main_services`)

## 6) Ввод данных пациента

После выбора услуги бот просит строку формата:

`Фамилия Имя Отчество, ДД.ММ.ГГГГ, телефон`

Парсер разбирает:
- фамилию / имя / отчество
- дату рождения (`YYYY-MM-DD`)
- телефон

## 7) Экран подтверждения

Бот показывает карточку проверки:
- ФИО пациента
- телефон
- дата рождения
- услуга
- доктор
- филиал
- дата и время приема

Кнопки:
- `Подтвердить`
- `Назад` (вернуться к вводу пациента)

## 8) Создание записи в МИС (после подтверждения)

Только после `Подтвердить` вызывается создание записи.

Метод МИС:
- `BookAnAppointmentWithParams` через эндпоинт:
  - `.../hs/bwi/AppointmentCreate` (основной)
  - fallback: `.../hs/bwi/BookAnAppointmentWithParams`
- важный параметр: `EmployeeID` (для текущего контура)
- также передается: `Service` (выбранная услуга), если выбрана.

## 9) Локальное сохранение и финальный ответ

После успешной МИС-записи бот:
- сохраняет запись в таблицу `appointments` (`mis_uid`, пациент, дата/время, филиал, услуга),
- чистит временные поля сценария из сессии,
- возвращает в состояние `schedule_ready`,
- отправляет подтверждение пользователю.
